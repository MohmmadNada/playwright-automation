# Playwright Automation Framework

[![Playwright Tests](https://github.com/MohmmadNada/playwright-automation/actions/workflows/playwright.yml/badge.svg)](https://github.com/MohmmadNada/playwright-automation/actions/workflows/playwright.yml)

## Overview

A scalable end-to-end test automation framework built with [Playwright](https://playwright.dev/) and TypeScript. Covers two test targets:

- **UI** — [SauceDemo](https://www.saucedemo.com): login, product sorting, and checkout flows
- **API** — [Simple Books API](https://simple-books-api.click): full CRUD lifecycle for orders

**Tech stack:**

| Layer | Tool |
|---|---|
| Test runner | Playwright (latest) |
| Language | TypeScript (strict mode) |
| Browsers | Google Chrome, Mozilla Firefox |
| CI/CD | GitHub Actions |
| Reports | Playwright HTML Reporter |
| Node.js | v22 LTS |

---

## Prerequisites

- [Node.js v22 LTS](https://nodejs.org/) — use `nvm use` if you have [nvm](https://github.com/nvm-sh/nvm) installed (`.nvmrc` is included)
- npm (bundled with Node.js)

## Setup & Installation

1. Clone the repository:
   ```bash
   git clone https://github.com/MohmmadNada/playwright-automation.git
   cd playwright-automation
   ```

2. Switch to the correct Node.js version:
   ```bash
   nvm use
   ```

3. Install dependencies:
   ```bash
   npm ci
   ```

4. Install Playwright browsers and system dependencies:
   ```bash
   npx playwright install --with-deps
   ```

## Running Tests

```bash
# Run all tests (Chrome + Firefox for UI, API project)
npm test

# Run UI tests only
npm run test:ui

# Run API tests only
npm run test:api

# Run with visible browser (headed mode)
npm run test:headed

# Run a single test by ID
npx playwright test --grep TC_UI_001

# Run a single spec file
npx playwright test tests/ui/login.spec.ts

# Type-check without running tests
npm run typecheck
```

## Architecture

```
playwright-automation/
├── src/
│   ├── api/            # BooksApiClient — wraps all HTTP calls to Simple Books API
│   ├── constants/      # Shared constants (e.g. HttpStatus)
│   ├── data/           # Test data JSON files (ui-test-data.json, api-test-data.json)
│   ├── fixtures/       # Custom Playwright fixture extensions (DI layer)
│   ├── pages/          # Page Object classes — one file per page
│   ├── types/          # Shared TypeScript interfaces (api.types.ts)
│   └── utils/          # Pure helper functions (randomEmail, sumPrices, etc.)
├── tests/
│   ├── ui/             # UI spec files
│   └── api/            # API spec files
├── .github/workflows/  # GitHub Actions CI/CD pipeline
├── global-setup.ts     # Suite-level setup: SauceDemo login → storageState.json
├── global-teardown.ts  # Suite-level teardown: logs suite end time
├── playwright.config.ts
├── tsconfig.json
└── .nvmrc              # Node.js version pin (v22 LTS)
```

**Call flow:**

```
test file
  → imports test from src/fixtures/*.ts    ← custom fixture, not raw Playwright
  → fixture injects src/pages/* or src/api/*
  → page/client reads src/data/*.json for inputs
  → dynamic values generated by src/utils/helpers.ts
```

**Key design decisions:**

- **Page Object Model** — all browser interactions are encapsulated in page classes; spec files only call page methods and make assertions
- **API Client pattern** — `BooksApiClient` owns all HTTP calls and returns typed responses; tests never call `request.get/post/patch/delete` directly
- **Global auth** — `global-setup.ts` performs a single headless login and saves the session to `storageState.json`; UI tests that need an authenticated state consume it via fixture instead of re-logging in
- **Dependency injection via fixtures** — tests declare what they need in their function signature; fixtures provide it

## Viewing Reports

After a test run, open the HTML report locally:

```bash
npx playwright show-report reports
```

The report includes test results, error details, traces, and screenshots on failure.

## CI/CD

The pipeline is defined in `.github/workflows/playwright.yml` and runs on two triggers:

| Trigger | When |
|---|---|
| Push to `main` | Every merge to the main branch |
| Scheduled | Daily at 03:30 UTC (06:30 Amman, UTC+3) |

**Pipeline steps:**

1. Checkout repository
2. Set up Node.js v22 (read from `.nvmrc`)
3. `npm ci` — install dependencies from lockfile
4. `npx playwright install --with-deps` — install browsers + OS-level dependencies
5. `npm test` — run the full test suite
6. Upload `reports/` as a GitHub Actions artifact (retained for 30 days, uploaded even on failure)

**Artifact location:** GitHub → Actions tab → select a workflow run → *Artifacts* section → `playwright-report-<run-id>`.
